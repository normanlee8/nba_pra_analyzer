The following updates to the meta-modeling layer transition the system from a static heuristic-based approach to a robust, time-aware machine learning framework designed for real-world betting environments. By implementing a chronological data split and a 45-day rolling training window in meta_training.py, the system eliminates "data leakage" where future game results could inadvertently influence past predictions, ensuring that the model only learns from information that would have been available at the time of the bet. The integration of CalibratedClassifierCV using isotonic regression addresses the inherent bias in XGBoost probability outputs, transforming raw model scores into true hit-rate probabilities that accurately reflect historical win percentages. Additionally, the inference logic in inference.py was refined to handle missing features using np.nan instead of zero-filling; this allows the XGBoost algorithm to utilize its native "sparsity-aware" branching to route incomplete data down the most statistically appropriate paths rather than forcing them into extreme "zero" values. Together, these changes resolve issues of over-optimization and "concept drift," creating a self-correcting feedback loop that can reliably identify "traps" and "hidden edges" based on the primary model's current performance trends.
