The updates to prop_analyzer/data/scraper.py implement a multi-layered caching and bypass system designed to eliminate redundant network requests and processing overhead. The primary change involves a significant overhaul of the should_skip_season_file function, which now enforces a strict "skip" rule for completed past seasons (2024-25) and a dynamic 8-hour refresh window for current season data. This directly addresses the inefficiency of re-downloading static historical data and prevents the script from hitting external servers like TeamRankings multiple times a day when data has not changed. Additionally, the scrape_espn_box_scores_incremental function was modified with a "past-season bailout" to stop the script from re-checking empty dates, such as the All-Star break, where no games occurred. Finally, the caching logic was integrated into the PBPStats scraping process, which previously ran unconditionally. These changes collectively solve the latency issues by ensuring the scraper only targets active, missing, or stale data, reducing the execution time for historical seasons to nearly zero and optimizing live season updates.
