To eliminate historical target leakage and improve the model’s predictive power, the system was transitioned from using static daily snapshots to a point-in-time, time-series architecture for injury and Play-By-Play (PBP) data. The primary issue addressed was a "time-travel" bug where the model retroactively applied today’s injury reports and season-total PBP metrics—such as Assist Network shares and With/Without You (WOWY) efficiency—to games played months ago, which artificially inflated accuracy during training but caused failures in live environments. By modifying the scraper to maintain a historical injury ledger and updating the feature generator to utilize pd.merge_asof, the system now ensures that each historical game is only evaluated against the specific rotation and vacancy data that existed at that exact moment in time. These changes solve the leakage problem by grounding all advanced PBP features—including foul trouble risk, usage absorption, and lineup-based efficiency shifts—in chronological reality, thereby providing a more robust and realistic foundation for the Bayesian PRA projections.
